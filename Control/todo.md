# 码风
1. 建议文件末尾添加换行符（空行）
2. 已上传我个人风格的.clang-format，可根据自己码风修改
3. 头文件守卫尽量不以下划线开头，考虑使用 `#pragma once`

# 工程架构
1. 建议添加 `config.h` ，存储宏开关和频繁更改的参数并添加注释
2. 多个文件共享，但不常修改的宏定义和别名，也存入单独的头文件
3. 个人习惯，**PID** 计算直接封装成一个 `PID_Control()` 函数供外部调用，内部再放入目前的 `chassis.c` 相关逻辑。二者没有绝对优劣之分，视工程和个人习惯而定
4. 非必需情况，尽量减少代码块内部的静态变量，而是在全局变量管理并考虑是否加 `static` 修饰

# 性能优化
1. **STM32F103** 系列单片机基于 **Cortex-M3** 内核，未配备硬件 FPU 。建议减少浮点数使用，如有能力，可尝试将全部浮点运算重构为特定定点数（视性能开销而定，只是从性能优化角度建议）
2. 语句量较少的函数可以考虑将实现放入头文件并添加 `__STATIC_INLINE` 修饰
3. 临时变量和各种函数的整型传入值，返回值，建议使用 **32 位**整型
